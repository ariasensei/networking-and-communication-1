Lesson 5

Network Math: Understanding Hosts and Addresses

Network math helps technicians estimate how many devices can join a network. The goal is not to memorize every number immediately. Start by understanding that an IP address has network bits and host bits, and the host bits identify individual devices.

USABLE HOSTS = 2^(HOST BITS) − 2

Reading an IPv4 Address

An IPv4 address has four number groups called octets. For example, 192.168.1.25 has four octets. A subnet mask or prefix, such as /24, tells us which part belongs to the network and which part is available for host devices.

192.168.1.25

Example IPv4 address: identifies one device on a network.

/24

Prefix length: 24 bits are for the network, leaving 8 bits for hosts.

Worked Example: A /24 Network

  1. 01A /24 prefix leaves 32 − 24 = 8 host bits.
  2. 02Use the formula: 2⁸ − 2 = 256 − 2.
  3. 03The answer is 254 usable host addresses. The two excluded addresses are the network address and broadcast address.

Lesson 6

Evolution of Computer Networking

Computer networking developed from research projects that connected a small number of large computers. Today, networking connects billions of personal devices, servers, sensors, classrooms, businesses, and cloud services.

A Simple Networking Timeline

These milestones show how networks changed from experimental links into the everyday infrastructure we use now.

1960s

Early Packet-Switched Networks

Researchers explored sending information in smaller packets instead of reserving one full communication line.

1970s–1980s

TCP/IP and the Internet

TCP/IP became a common set of rules that allowed different networks to communicate with one another.

1990s

The World Wide Web Expands

Web browsers made information easier to access, helping the internet become useful for schools, homes, and businesses.

Today

Mobile, Cloud, and Smart Networks

Wi-Fi, mobile networks, cloud services, and connected devices allow people to communicate and work from almost anywhere.

Why This History Matters

Modern networks still use the same big idea: devices need agreed rules, reliable paths, and clear addresses so data can reach the correct destination. The tools became faster and more wireless, but the communication problem is still the same.

Network Connectivity Devices

A network works because devices have different jobs. Some connect computers within one room, some provide Wi-Fi, some connect separate networks, and some connect the school network to an internet provider.

Hub

A hub connects devices, but it sends incoming data to every connected port. It is simple but less efficient than a switch.

Remember the Main Difference

A switch usually connects devices within one local network. A router connects different networks. An access point lets wireless devices join the local network.

Lesson 8

Network Topology: How Devices Are Arranged

Network topology describes the arrangement of devices and connections. Choosing the right topology affects cost, performance, troubleshooting, and what happens if one cable or device fails.

Select a topology to compare its purpose, advantage, and limitation.

Topology explorer

Star Topology

In a star topology, every device has its own connection to a central switch or hub. This is one of the most common designs in schools and offices.

SW
PC
PC
PC
AP

Strength

Easy to expand and troubleshoot because each device has a separate link.

Caution

If the central switch fails, all connected devices lose their network connection.

Which Topology Fits a Computer Laboratory?

For most modern school laboratories, a star topology is a practical choice. Each computer connects to a central switch. If one computer cable fails, the other computers can usually continue working. The central switch should be protected because it is an important point of connection.

Laboratory Activity 2

Network Classifications and Architecture

In this laboratory activity, you will identify the size or classification of networks and create a labeled network architecture diagram. Follow each step carefully and submit a clear output according to your instructor’s directions.

Clear Instructions

  1. 01Review the classifications: PAN, LAN, MAN, and WAN. Write one real-life example for each classification, such as Bluetooth earbuds for PAN or a campus system for LAN.
  2. 02Choose one setting to design: a computer laboratory, a small office, a library, or a classroom with Wi-Fi. List the users, devices, and services that need network access.
  3. 03Draw a network architecture diagram on paper or in a digital tool. Start with the internet, then add the modem or ONT, router, switch, access point, and end devices in the correct order.
  4. 04Label every major device and choose a topology. For a standard computer laboratory, use a star topology and place the switch at the center of the layout.
  5. 05Write a short explanation below the diagram. State the network classification, topology, the purpose of each connectivity device, and one reason why your design is appropriate.

Expected output

One Labeled Architecture Diagram

Your output must include: network classification, chosen topology, at least four connectivity devices or end-device groups, labels for connection paths, and a brief written explanation. Keep your diagram clean enough for another person to follow.
