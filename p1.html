Lesson 1

Introduction to Computer Networking

A computer network is a group of two or more connected devices that can communicate and share resources. The devices may be computers, phones, printers, servers, cameras, or smart devices. Connections can be wired through cables or wireless through radio signals such as Wi-Fi.

Why Networks Matter

  • Resource sharing: many users can use the same printer, files, internet connection, or software service.
  • Communication: people can send messages, join video meetings, and collaborate on documents.
  • Centralized services: a school can provide shared learning systems, backups, and user accounts.

Realistic Scenario: School Enrollment Day

During enrollment, staff in different offices need access to the same student information system. Their PCs connect through a local network to a server or cloud service. If the network is unavailable, staff may not be able to verify records, print forms, or update student information.

Common Network Types

Networks are often described by the area they cover.

PAN

Personal Area Network

A very short-range network, such as a phone connected to Bluetooth earbuds.

LAN

Local Area Network

A network in a home, classroom, office, or computer laboratory.

MAN

Metropolitan Area Network

A network that connects locations across a city or municipality.

WAN

Wide Area Network

A network that connects distant locations, such as the internet.

Wired Communication

Wired communication sends data through a physical medium, such as Ethernet cable or fiber optic cable. It is common for desktop PCs, switches, printers, and servers because it can provide stable and fast connections.

Realistic Example

In a school computer laboratory, desktop PCs often use Ethernet cables connected to a switch. This reduces Wi-Fi congestion when many students are online at the same time.

Protocols: Rules for Communication

Just as people need a common language and agreed turn-taking rules, network devices need protocols. Protocols define how a message is formatted, sent, received, checked, and acknowledged. Examples include IP for addressing, TCP for reliable delivery, and HTTP/HTTPS for web pages.

Lesson 3

Using a PC on a Network

A PC becomes part of a network when it has a network adapter, a valid connection, and the correct network settings. Once connected, the PC can communicate with other devices and access local or online services.

How a PC Connects

  1. 01Network interface card (NIC): the PC uses a wired Ethernet adapter or wireless adapter to connect.
  2. 02Connection: the PC joins through an Ethernet cable to a switch or through Wi-Fi to an access point.
  3. 03Addressing: the PC receives an IP address, usually automatically from a router or DHCP server.
  4. 04Access: the PC can now reach local services, printers, file shares, and the internet if permitted.

Troubleshooting Scenario

A student in the computer lab cannot open the learning platform, while other students can. The technician should check the physical cable or Wi-Fi signal, network icon status, IP settings, and whether the switch port is active before assuming the website is the problem.

Troubleshooting order

Check power → check cable/Wi-Fi → check network status → check IP address → test another website or service.

Lesson 4

Computer Hardware Basics

Computer hardware refers to the physical parts of a computer. Understanding hardware helps you troubleshoot devices, upgrade systems, and identify what is required for a PC to connect reliably to a network.

CPU

The central processing unit executes instructions and processes data. A faster CPU can improve performance when many applications run at once.

RAM

Random access memory stores data temporarily while programs are running. Insufficient RAM can make a PC slow during research or online meetings.

Storage

SSD and HDD drives keep the operating system, applications, and files even when power is off. SSDs generally load systems more quickly.

NIC / Wi-Fi Adapter

The network interface connects the computer to wired or wireless networks and is essential for online communication.

Realistic Scenario: Preparing a Lab PC

Before a computer laboratory can run a web-based assessment, each PC must have enough memory to run the browser, working storage for the operating system, an active network adapter, and a reliable monitor, keyboard, and mouse. A damaged Ethernet port or disabled Wi-Fi adapter can prevent only one PC from joining the network even when the rest of the lab is online.

Hardware and networking are connected: a network cannot function correctly if the end devices have failed ports, damaged cables, missing drivers, or insufficient power.
